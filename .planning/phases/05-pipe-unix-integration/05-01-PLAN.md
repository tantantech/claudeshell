    - runPipe("") with empty stdin writes "No input provided" to stderr and exits 1
    - runPipe() creates renderer with isTTY: false (per D-08/D-09)
    - runPipe() calls renderCostFooter on stderr after AI response (per D-10)
    - runPipe() exits process with code 0 on success, 1 on error (per D-07)

  <action>
    Create `src/pipe.ts` with two exported functions per D-01 through D-12:

    1. `collectStdin()`: async function that reads `process.stdin` via `for await` loop.

       - After collection, check `content.includes('\0')` for binary rejection.
       - If `totalBytes > WARN_STDIN_BYTES` (102_400), write warning to `process.stderr`.
       - Return the UTF-8 string.

    2. `runPipe(cliPrompt: string)`: async function that:
       - Calls `collectStdin()` to get stdin content
       - Per D-05: if `cliPrompt` is non-empty, combine as `${cliPrompt}\n\n---\n\n${stdinContent}`
       - Per D-06: if `cliPrompt` is empty, use `stdinContent` as the prompt

       - Create renderer via `createRenderer({ isTTY: false })`
       - Create AbortController
       - Call `executeAI(prompt, { cwd: process.cwd(), lastError: undefined, abortController, callbacks: { onText: renderer.onText, onToolStart: renderer.onToolStart, onToolEnd: renderer.onToolEnd, onError: (msg) => process.stderr.write(msg + '\n') } })`
       - Call `renderer.finish()`
       - If `result.usage`, call `renderCostFooter(result.usage)`
       - Call `process.exit(0)`
