Per D-26, D-27, D-28, D-29, replace the static `BUNDLED_PLUGINS` array with a lazy-loading system:

1. **Keep backward-compatible `BUNDLED_PLUGINS`** export but change it to only contain the original 16 plugins (for any code that still references it synchronously). Mark with a comment: `// Legacy: use loadBundledPlugins() for full catalog`.

2. **Add `PLUGIN_CATEGORY` lookup table**: `Record<string, string>` mapping plugin name to category directory. Built from PLUGIN_CATALOG. Example: `{ 'kubectl': 'aliases', 'colored-man-pages': 'hooks', 'git': '.', 'extract': 'utilities', ... }`. Note: existing plugins in the root plugins/ dir use category `'.'`, completions use `'completions'`, utilities use `'utilities'`.

3. **Add `PLUGIN_CATALOG_LIST`**: Lightweight array of `{ name, description, category }` for search. Derived from PLUGIN_CATALOG, filtered to `status !== 'no-equivalent'`.

4. **Add `loadBundledPlugins(enabled: readonly string[])`**: Async function that:
   - Groups enabled plugin names by category using PLUGIN_CATEGORY
   - For each category, uses `await import()` to dynamically load only the needed modules
   - Returns `readonly PluginManifest[]`
   - Falls back to BUNDLED_PLUGINS entries for the original 16 plugins (root-level files)
   - Per pitfall 5, the lookup table is a plain object literal -- no circular imports

```typescript
export async function loadBundledPlugins(
  enabled: readonly string[]
): Promise<readonly PluginManifest[]> {
  const plugins: PluginManifest[] = []
  
  for (const name of enabled) {
    const category = PLUGIN_CATEGORY[name]
    if (!category) continue
    
    try {
      if (category === '.') {
        // Root-level plugin (original 16)
        const existing = BUNDLED_PLUGINS.find(p => p.name === name)
        if (existing) plugins.push(existing)
      } else {
        const mod = await import(`./${category}/${name}.js`) as { plugin: PluginManifest }
        plugins.push(mod.plugin)
      }
    } catch {
      // Plugin file missing or broken — skip silently, loader handles errors
    }
  }
  
  return plugins
}
```
