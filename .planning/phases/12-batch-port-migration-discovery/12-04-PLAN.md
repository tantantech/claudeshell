1. **MigrationStatus interface:**
```typescript
export interface MigrationStatus {
  readonly omzName: string
  readonly neshEquivalent: string | null
  readonly status: 'available' | 'partial' | 'missing'
}
```

2. **detectOMZ():** Check `existsSync(join(homedir(), '.oh-my-zsh'))`. Per pitfall 7, also verify .zshrc has `plugins=` line -- if directory exists but no plugins line, return false.

3. **parseZshrcPlugins(content: string):** Per pitfall 2, use multi-line regex: `/^plugins=\(\s*([\s\S]*?)\s*\)/m`. Split on whitespace, filter empty strings and comments (lines starting with `#`). Return `readonly string[]`.

4. **parseZshrcFile():** Convenience that reads `~/.zshrc` and calls parseZshrcPlugins.

5. **generateMigrationReport(omzPlugins: readonly string[]):** Import PLUGIN_CATALOG from catalog.ts. For each OMZ plugin name, look up by `omzName` field. Map to MigrationStatus with appropriate status. Handle synonyms: OMZ `git` maps to Nesh `git` (aliases) -- the catalog has the mapping.

6. **formatMigrationReport(report: readonly MigrationStatus[]):** Per D-11, format with picocolors: green for available, yellow for partial, red for missing. Returns string for stdout.

Create `tests/migration.test.ts` testing parseZshrcPlugins with both single-line and multi-line formats, generateMigrationReport with known catalog entries, and edge cases (empty, comments, no match).

Create `src/migration/discovery.ts` per D-14 through D-19:

1. **DiscoveryResult interface:**
```typescript
export interface DiscoveryResult {
  readonly recommendations: readonly PluginRecommendation[]
  readonly source: 'ai' | 'keyword'
}

export interface PluginRecommendation {
  readonly name: string
  readonly description: string
  readonly reason: string
  readonly category: string
}
```

2. **buildDiscoveryPrompt(userInput, catalog):** Per research pattern 6, embed the catalog as `name | description | category` lines. The ~300 entries fit in ~3,750 tokens. System prompt tells Claude to return 3-5 most relevant plugins as a ranked list.

3. **discoverPlugins(userInput: string):** 
   - Load config via loadConfig() to check for API key
   - Per D-19, if no API key: fall back to keyword-based search (filter PLUGIN_CATALOG where name or description includes any word from userInput). Return `{ recommendations, source: 'keyword' }`.
   - If API key exists: Per D-15, use a lightweight AI call. Per pitfall 6, do NOT reuse executeAI (which enables tool-use). Instead, use `@anthropic-ai/claude-agent-sdk` directly with `allowedTools: []` to disable all tools. Use `claude-3-5-haiku` model for fast, cheap responses.
   - Parse the AI response to extract plugin names and match against catalog.
   - Return `{ recommendations, source: 'ai' }`.

4. **formatDiscoveryResults(result: DiscoveryResult):** Per D-17, format as ranked list with picocolors. Show plugin name (bold), description, reason, and enable command (`plugin enable <name>`).

Important: The AI call should be wrapped in try/catch. If the AI call fails (network error, rate limit, etc.), fall back to keyword search silently. Never crash.

Import path for SDK: use lazy `await import('@anthropic-ai/claude-agent-sdk')` to avoid loading at module import time (same lazy-load pattern as src/ai.ts).
