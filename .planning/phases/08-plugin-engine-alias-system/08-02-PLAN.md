  <action>
    Create `src/plugins/resolver.ts` (per D-20, PLUG-05):

    Implement `topologicalSort(plugins: readonly PluginManifest[]): { readonly sorted: readonly PluginManifest[]; readonly cycles: readonly string[] }` using Kahn's algorithm:
    1. Build `nameMap: Map<string, PluginManifest>` from plugins
    2. Build `inDegree: Map<string, number>` initialized to 0 for each plugin
    3. Build `adjacency: Map<string, string[]>` — for each plugin's dependencies, if dep exists in nameMap, add edge from dep to plugin and increment inDegree
    4. Initialize queue with all plugins having inDegree === 0
    5. Process queue: dequeue, add to sorted, decrement neighbors' inDegree, enqueue at 0
    6. Any plugins NOT in sorted are in cycles — return their names in `cycles` array
    7. Log cycles to stderr: `[nesh] plugin dependency cycle detected: ${cycles.join(', ')}\n`

    Create `src/plugins/hooks.ts` (per D-24 through D-27, PLUG-08):

    Implement `dispatchHook(hookName: string, handlers: readonly HookHandler[], context: Readonly<HookContext>): Promise<void>`:
    1. If handlers is empty, return immediately
    2. Run `Promise.allSettled(handlers.map(h => Promise.resolve(h(context))))`
    3. For each rejected result, write to stderr: `[nesh] hook "${hookName}" handler failed: ${reason.message}\n`

    Implement `buildHookBus(plugins: readonly PluginManifest[]): HookBus`:
    - Type `HookBus = Readonly<Record<HookName, readonly HookHandler[]>>`
    - Iterate plugins, collect each plugin's `hooks.preCommand`, `hooks.postCommand`, etc. into arrays
    - Return frozen object with all four hook arrays

    Write tests using vi.spyOn(process.stderr, 'write') for error assertions.
