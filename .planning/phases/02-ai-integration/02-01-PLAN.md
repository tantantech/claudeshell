  <action>
    **src/passthrough.ts** -- Per D-24, D-25, implement stderr capture with tee pattern:

    1. Import `LastError` is NOT needed here -- just return a new `CommandResult` type.

    2. Add and export a new interface at the top:
       ```typescript
       export interface CommandResult {
         readonly exitCode: number
         readonly stderr: string
       }
       ```

    3. Change `executeCommand` return type from `Promise<number>` to `Promise<CommandResult>`.

    4. Change spawn stdio from `'inherit'` to `['inherit', 'inherit', 'pipe']` -- this pipes stderr only while keeping stdin/stdout inherited.

    5. Add stderr tee pattern:
       ```typescript
       const stderrChunks: Buffer[] = []
       child.stderr?.on('data', (chunk: Buffer) => {
         stderrChunks.push(chunk)
         process.stderr.write(chunk)  // tee: show in real-time
       })
       ```

    6. In the `close` handler, resolve with:
       `{ exitCode: code ?? 1, stderr: Buffer.concat(stderrChunks).toString('utf-8') }`

    7. In the `error` handler, resolve with:
       `{ exitCode: 127, stderr: err.message }`

    4. Test: `executeCommand('command_that_does_not_exist_xyz')` returns exitCode non-zero
