    **src/renderer.ts** -- Per D-06, D-07, D-08, D-09, D-10, D-11, D-12, D-13, D-14:

    Create a streaming renderer factory function:

    ```
    export interface Renderer {
      readonly onText: (text: string) => void
      readonly onToolStart: (toolName: string) => void
      readonly onToolEnd: (toolName: string, result?: string) => void
      readonly finish: () => void
    }

    export function createRenderer(options: { isTTY: boolean }): Renderer
    ```

    Implementation details:
    1. Accumulate text chunks in an array (for final markdown rendering)
    2. `onText`: call `process.stdout.write(text)` immediately for streaming (D-07). Also push to accumulator.
    3. `onToolStart(name)`: write to `process.stderr` using picocolors `dim()`:
       - Format: `dim('\n  -> Using ${name}...')` when TTY (D-11, D-12)
       - Write nothing when not TTY
       - Track `inTool = true`
    4. `onToolEnd(name, result?)`: write to `process.stderr`:
       - Format: `dim(' done\n')` when TTY
       - If result provided, write `dim('     ${result}\n')` (D-13)
       - Set `inTool = false`
    5. `finish()`:
       - Write `'\n'` for spacing after response (D-09)
       - NOTE on markdown rendering: Per Pitfall 5 in RESEARCH.md, rendering markdown on streaming fragments breaks formatting. For v1, stream raw text for responsiveness. The `finish()` method does NOT re-render -- the raw streamed text is the output. Markdown rendering can be enhanced in v2 with paragraph-boundary detection. This keeps v1 simple and working.
    6. Import `picocolors` for dim/gray styling (D-08, already installed)
    7. When `isTTY` is false (D-10): `onText` writes plain text, tool indicators are suppressed, `finish` adds newline only

    **src/ai.ts** -- Per D-01, D-02, D-03, D-04, D-20, D-21, D-22, D-26:

    Create the AI execution module:

    ```
    export interface AICallbacks {
      readonly onText: (text: string) => void
      readonly onToolStart: (toolName: string) => void
      readonly onToolEnd: (toolName: string, result?: string) => void
      readonly onError: (message: string) => void
    }

    export async function executeAI(
      prompt: string,
      options: {
        cwd: string
        lastError: LastError | undefined
        abortController: AbortController
        callbacks: AICallbacks
      }
    ): Promise<void>
    ```

    Implementation details:
    1. **Lazy-load SDK** (D-01): Use a module-level `let sdkModule` variable. On first call, `sdkModule = await import('@anthropic-ai/claude-agent-sdk')`. Cache for subsequent calls.

    2. **Check API key** (D-15, D-27, D-28, D-29): Call `resolveApiKey()` from `./config.js`. If undefined, call `callbacks.onError('Set ANTHROPIC_API_KEY to use AI commands. Example: export ANTHROPIC_API_KEY=sk-ant-...')` and return early.

    3. **Build prompt with context** (D-26): If `prompt` is `'explain'` or `'why'` AND `options.lastError` exists, prepend error context:
       ```
       `The following command failed:\n\`${lastError.command}\`\nExit code: ${lastError.exitCode}\nStderr:\n${lastError.stderr}\n\nExplain what went wrong and how to fix it.`
       ```
       Otherwise use the prompt as-is.

    4. **Build system prompt** (Claude's discretion): Include cwd, OS platform, Node version:
       ```
       `You are ClaudeShell, an AI assistant running inside a terminal shell.\nCurrent directory: ${cwd}\nOS: ${process.platform}\nNode: ${process.version}`
       ```

    5. **Show thinking indicator**: Before starting the SDK query, write `dim('Thinking...\r')` to stderr. Clear it when first text_delta arrives by writing `'             \r'` (overwrite with spaces).

    6. **Call SDK query()** (D-02, D-03):
       ```typescript
       const stream = sdkModule.query({
         prompt: fullPrompt,
         options: {
           abortController: options.abortController,
           includePartialMessages: true,
           allowedTools: ['Read', 'Write', 'Edit', 'Bash', 'Glob', 'Grep'],
           permissionMode: 'acceptEdits',
           cwd: options.cwd,
           systemPrompt: systemPrompt,
         }
       })
       ```

    7. **Process stream events**: Iterate with `for await (const message of stream)`:
       - If `message.type === 'stream_event'`:
         - `event.type === 'content_block_start'` with `event.content_block.type === 'tool_use'`: call `callbacks.onToolStart(event.content_block.name)`
         - `event.type === 'content_block_delta'` with `event.delta.type === 'text_delta'`: call `callbacks.onText(event.delta.text)`. Clear thinking indicator on first text.
         - `event.type === 'content_block_stop'`: call `callbacks.onToolEnd('')`
       - If `message.type === 'assistant'` and `message.error`: call `callbacks.onError(mapSDKError(message.error))`
       - If `message.type === 'result'` and `message.is_error`: call `callbacks.onError('AI request failed')`

    8. **Error classification** (D-15 through D-19): Create internal `classifyError(error: unknown): string`:
       - Contains 'api key' or 'authentication' -> `'Invalid API key -- check ANTHROPIC_API_KEY'`
       - Contains 'rate limit' -> `'Rate limited -- wait a moment and try again'`
       - Contains 'network' or 'ECONNREFUSED' or 'ENOTFOUND' -> `'Network error -- check your connection'`
       - Contains 'billing' -> `'Billing error -- check your Anthropic account'`
       - Default: `'AI error: ${message}'`

    9. **Top-level try/catch** (D-20): Wrap everything in try/catch. On abort (AbortError or abort signal), do NOT call onError -- the shell handles the [cancelled] message. On other errors, call `callbacks.onError(classifyError(error))`.
