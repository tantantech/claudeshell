  <name>Task 1: Segment collectors and TTL cache</name>
  <files>src/segments.ts, src/segment-cache.ts, tests/segments.test.ts, tests/segment-cache.test.ts</files>
  <behavior>
    - SegmentCache: set/get with TTL expiry (default 2000ms), get returns undefined after TTL
    - SegmentCache: clear() empties all entries
    - getGitStatus(): returns { dirty, staged, untracked, ahead, behind, stash } counts by parsing `git status --porcelain=v2 --branch`; returns null outside git repo; result is cached
    - getExecTime(startMs, thresholdMs=1000): returns formatted duration string only if elapsed > threshold, e.g. "1.2s", "2m 15s"; returns undefined below threshold
    - getExitCode(code): returns code as string if non-zero, undefined if 0
    - getClock(): returns current time as "HH:MM"
    - getNodeVersion(): returns node major version string like "v22" by reading process.version
    - getPythonVenv(): returns basename of $VIRTUAL_ENV if set, undefined otherwise
    - getUserHost(): returns "user@hostname" from os.userInfo() and os.hostname()
  </behavior>
  <action>
    Create `src/segment-cache.ts` with a generic SegmentCache class:
    - Constructor takes default TTL in ms (default 2000)
    - `get(key)` returns cached value if within TTL, undefined otherwise
    - `set(key, value, ttlOverride?)` stores value with timestamp
    - `clear()` empties cache
    - Immutable: cache entries are readonly objects, Map is internal implementation detail
    - Export a singleton `segmentCache` instance with 2s default TTL

    Create `src/segments.ts` with all segment data collectors:
    - `getGitStatus()`: Run `git status --porcelain=v2 --branch` with 500ms timeout via execFileSync. Parse output for: modified (dirty) count from lines starting with "1 " or "2 ", staged from lines where index column is not ".", untracked from "? " lines, ahead/behind from `# branch.ab` header. Cache result using segmentCache with key=cwd. Return `GitStatusInfo | null`.
    - `getExecTime(startMs, thresholdMs=1000)`: calculate elapsed, format as "Xs" or "Xm Ys", return undefined if below threshold.
    - `getExitCode(code)`: return string of code if truthy, else undefined.
    - `getClock()`: return HH:MM from new Date().
    - `getNodeVersion()`: return process.version split to major (e.g. "v22").
    - `getPythonVenv()`: return path.basename of process.env.VIRTUAL_ENV or undefined.
    - `getUserHost()`: return `${os.userInfo().username}@${os.hostname()}`.

    Define `SegmentName` union type for all segment identifiers: 'shell-name' | 'dir' | 'git-branch' | 'git-status' | 'exec-time' | 'exit-code' | 'clock' | 'node-version' | 'python-venv' | 'user-host'.

    Define `SegmentData` interface: `{ readonly text: string; readonly icon?: string; readonly fgColor: number; readonly bgColor: number }`.

    Create test files with vitest covering: TTL cache expiry behavior, git status parsing from mock output (mock execFileSync), exec time formatting at various durations, exit code edge cases (0 vs non-zero), all simple collectors.
  </action>

    - `executePromptConfig(rl: readline.Interface): Promise<PromptConfigResult>` that walks user through:
      1. Icon mode selection: "Does this look like a diamond? (Nerd Font test)" → auto-detect or let user pick nerd-font/unicode/ascii
      2. Segment selection: show numbered list of all SegmentNames with current on/off state, user toggles by number, "done" to finish
      3. Preview the prompt with current selections before confirming
    - `PromptConfigResult` interface: `{ readonly segments?: readonly string[]; readonly iconMode?: 'nerd-font' | 'unicode' | 'ascii' }`
    - Export `getIcon(name: string, mode: 'nerd-font' | 'unicode' | 'ascii'): string` — maps icon names to glyphs per mode. Icons: branch (\uE0A0 / unicode branch symbol / ""), folder (\uF115 / unicode folder / ""), clock (\uF017 / unicode clock / ""), node (\uE718 / "N" / "N"), python (\uE73C / "Py" / "Py"), error (\uF071 / "!" / "!"), check (\uF00C / unicode check / "+"), stash (\uF01C / "*" / "*")
    - Export `getSeparator(mode: 'nerd-font' | 'unicode' | 'ascii'): { right: string; rightThin: string }` — powerline arrows for nerd-font, unicode triangles for unicode, "|" for ascii
    - Default segments: ['shell-name', 'dir', 'git-branch', 'git-status', 'exec-time', 'exit-code']

    **Add 4 new p10k themes** to `src/templates.ts`:
    - Extend the `builders` record and `TEMPLATES` array with:
    - **rainbow**: Multi-colored bg segments (blue, green, yellow, red, etc.), powerline separators, all segments enabled. Signature p10k look.
    - **lean**: No background colors, colored text only, thin separators or spaces. Two-line with right-aligned segments on line 1 (clock, node, venv) and prompt on line 2. Minimal and fast.
    - **classic-p10k**: Dark background segments (236/238/240 range), white/colored text, powerline separators. Similar to p10k's "Classic" preset.
    - **pure**: Ultra-minimal, single-line, dim gray text with colored directory and git info. Inspired by sindresorhus/pure. No backgrounds, no separators.
    - Each new builder calls segment collectors from `src/segments.ts` and uses `getIcon()`/`getSeparator()` from `src/prompt-config.ts` respecting the configured icon mode (load from config).
