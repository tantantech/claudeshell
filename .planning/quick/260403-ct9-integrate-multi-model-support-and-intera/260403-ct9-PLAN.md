    ```typescript
    export type ProviderName = 'claude' | 'openai' | 'gemini'

    export interface ProviderChunk {
      readonly type: 'text' | 'tool_start' | 'tool_end' | 'error' | 'result'
      readonly content: string
    }

    export interface ProviderOptions {
      readonly prompt: string
      readonly cwd: string
      readonly abortController: AbortController
      readonly model?: string
      readonly systemPrompt: string
      readonly sessionId?: string
      readonly permissionMode?: string
    }

    export interface AIProvider {
      readonly name: ProviderName
      query(options: ProviderOptions): AsyncGenerator<ProviderChunk>
    }
    ```

    **Step 3 — Create `src/providers/claude.ts`:**
    Extract the existing Claude SDK streaming logic from `src/ai.ts` into a provider that implements `AIProvider`. Keep the lazy `import('@anthropic-ai/claude-agent-sdk')` pattern. The `query()` method should yield `ProviderChunk` items by mapping the existing `SDKMessage` stream events. Include tool use support (tool_start/tool_end chunks), session resume via `buildResumeOptions`, and permission mode via `toSDKPermissionMode`. Resolve API key from `ANTHROPIC_API_KEY` env var or `config.keys.anthropic`.

    **Step 4 — Create `src/providers/openai.ts`:**
    Implement `AIProvider` using the `openai` npm package. Use `OpenAI` client with `chat.completions.create({ stream: true })`. Map streaming chunks to `ProviderChunk` (delta.content -> text chunks). Default model: `gpt-4o`. Supported model shorthands: `gpt4o` -> `gpt-4o`, `gpt4.5` -> `gpt-4.5-preview`, `o3` -> `o3`. Resolve API key from `OPENAI_API_KEY` env var or `config.keys.openai`. Tool use is NOT supported for OpenAI provider initially — only text streaming. If no API key found, yield an error chunk with helpful message.

    **Step 5 — Create `src/providers/gemini.ts`:**
    Implement `AIProvider` using `@google/generative-ai`. Use `GoogleGenerativeAI` client with `generateContentStream`. Map streaming chunks to `ProviderChunk`. Default model: `gemini-2.5-pro`. Supported model shorthands: `gemini-pro` -> `gemini-2.5-pro`, `gemini-flash` -> `gemini-2.5-flash`. Resolve API key from `GOOGLE_API_KEY` or `GEMINI_API_KEY` env var or `config.keys.google`. If no API key, yield error chunk.

  <action>
    **Step 1 — Create `src/keys.ts`:**
    Interactive key management module. Exports:

    - `runKeysMenu(rl: readline.Interface): Promise<void>` — Main interactive flow:
      1. Display current keys status (which providers have keys set, masked display showing last 4 chars: `sk-...a1b2`)
      2. Show menu:
         ```
         API Keys:
           anthropic: sk-ant-...x1y2
           openai:    (not set)
           google:    (not set)

         [1] Add/update key
         [2] Remove key
         [3] Done
         ```
      3. Option 1 — prompt for provider name (anthropic/openai/google), then prompt for key value. Save to `~/.nesh/config.json` under `keys` object using `loadConfig()` + `saveConfig()` (immutable update).
      4. Option 2 — prompt for provider name, remove that key from config.
      5. Option 3 — return.
      6. Loop back to menu after each action until user picks Done.

    - `maskKey(key: string): string` — Returns masked display: if key length > 8, show first 6 chars + `...` + last 4 chars. If shorter, show `****`.

    - `getKeyStatus(): Record<string, string | undefined>` — Returns current key status for each provider by checking env vars first, then config. Returns the raw key or undefined.

    Key resolution order (for display): env var (`ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, `GOOGLE_API_KEY`/`GEMINI_API_KEY`) then `config.keys[provider]`. Display shows "(env)" suffix if key comes from env var.

    Add `executeTheme` is already there. The `keys` builtin does not need a function here since it needs readline access — it will be handled directly in shell.ts.

After keys menu completes, reload config to pick up any changes: `const updatedConfig = loadConfig()` and update state accordingly (new keys are in config, provider may have changed).

    - `maskKey('short')` returns `'****'`
    - `maskKey('')` returns `'****'`
    - `getKeyStatus()` returns correct structure (mock env vars in test)
